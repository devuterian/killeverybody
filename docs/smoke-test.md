# 수동 스모크 테스트 (실제 Mac)

CI는 GUI 앱을 대신 눌러 주지 않으니, **릴리즈 전·큰 변경 후** 아래를 맥에서 한 번씩 확인합니다.

## 준비

- [ ] 테스트용으로만 써도 되는 사용자 계정이 있으면 더 안전합니다.
- [ ] 중요한 문서는 저장해 둡니다.

## GUI 앱만

- [ ] 메모, 미리보기 등 **닥 앱**을 몇 개 켭니다.
- [ ] killeverybody 실행 → **GUI 앱만** → **대상 수집** → 목록에 켠 앱이 보이는지 확인합니다.
- [ ] 메뉴 막대 전용 성격의 앱(또는 프리셋/메뉴바로 지정한 앱)은 **목록에 없거나** 예외대로 동작하는지 확인합니다.
- [ ] **강제 종료 실행** 전에 취소로 빠져 나올 수 있는지 확인합니다.

## 현재 사용자 프로세스 전체

- [ ] 범위를 바꾼 뒤 다시 **대상 수집** 했을 때 목록이 합리적인지 봅니다.
- [ ] `launchd`, `WindowServer` 같은 이름이 **목록에 없어야** 합니다 (denylist).

## 정책 JSON

- [ ] 설정에서 **정책보내기**로 파일을 저장합니다.
- [ ] **정책 가져오기**로 다시 넣었을 때 예외·메뉴바 번들 목록이 기대와 같습니다.

## 관리자 모드 (선택)

- [ ] 비밀번호 창이 뜨는지, 취소 시 아무 일도 없는지 확인합니다.

실패한 항목은 [Issues](https://github.com/devuterian/killeverybody/issues)에 OS 버전·앱 버전과 함께 적어 주세요.
